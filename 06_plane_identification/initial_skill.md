  
## Skill: 空港ライブカメラからの自動航空機識別システム  
  
このドキュメントでは、YouTubeのライブ配信から定期的にスナップショットを取得し、物体検出（YOLO）とマルチモーダルAI（Qwen2.5-VL）を組み合わせて航空機の機種を判定する自動化フローを定義します。  
## 1. ワークフロー概要  
1. **データ取得**: streamlink または yt-dlp を使用し、10分間、60秒間隔でフレームをキャプチャ。  
2. **物体検出 (YOLO)**: キャプチャした画像から plane クラスを検出し、バウンディングボックスで切り抜き（Crop）。  
3. **視覚的理解 (Qwen2.5-VL)**: 切り抜いた画像を Qwen2.5-VL に入力し、機体記号（レジ番号）や機種名を詳細判定。  
4. **結果保存**: 判定結果を JSON または CSV 形式で保存。  
  
  
## 3. 実装ステップ  
## ステップ 1: フレームのキャプチャ  
OpenCVとStreamlinkを組み合わせて、指定した間隔でスナップショットを保存します。  
## ステップ 2: YOLOによる機体検出とクロップ  
最新の YOLOv8 または YOLOv10 を使用して、画像内の「航空機」のみを抽出します。背景（滑走路や建物）を排除することで、Qwen2.5-VL の認識精度を高めます。  
## ステップ 3: Qwen2.5-VL による詳細判定  
クロップされた画像を Qwen2.5-VL に渡し、以下のプロンプトで推論を行います。  
* **Prompt**: "Identify the aircraft model and airline in this image. If visible, provide the registration number."  
##   
  
## 5. 注意事項とTips  
* **解像度の確保**: YouTubeの配信設定を最高画質（1080p以上）に固定して取得してください。  
* **計算リソース**: Qwen2.5-VL は高いVRAMを要求するため、ローカル実行が難しい場合は API または Quantized（量子化）モデルの検討を推奨します。  
* **著作権**: ライブ配信の規約を確認し、個人利用の範囲内で実行してください。
